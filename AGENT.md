# AGENT.md — 项目文档索引

本文件为 AI 助手的工作指引，列出项目中各关键文档的路径与用途。

## 文档清单

| 文件 | 用途 |
|------|------|
| `SPEC.md` | **需求规格**。定义了数据模型、界面布局、渲染规则、交互行为、性能约束和验收用例，是所有开发工作的唯一需求来源。 |
| `PLAN.md` | **开发计划**。将 SPEC 拆解为 7 个阶段共 28 个任务，包含技术栈选型、阶段依赖关系、每个任务对应的 SPEC 章节引用和预期产出文件。 |
| `PROGRESS.md` | **进度跟踪**。记录每个任务的完成状态（🔲/🔶/✅/❌），开发过程中需同步更新。底部有变更记录表。 |
| `AGENT.md` | **本文件**。项目文档索引与工作约定。 |

## 工作约定

1. 开始任务前，先阅读 `SPEC.md` 中对应章节确认需求细节。
2. 完成任务后，更新 `PROGRESS.md` 中对应条目的状态和备注。
3. 遇到需求歧义时，以 `SPEC.md` 为准。
